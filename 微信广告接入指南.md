# 微信小游戏广告接入指南

## 一、接入前准备

### 1.1 开通流量主

在使用广告功能前，需要先开通微信小游戏流量主。

**开通条件**：
- 累计独立访问用户（UV）不低于 1000
- 小游戏类目符合要求（非违规类目）
- 小游戏已发布上线

**开通流程**：
1. 登录微信公众平台（https://mp.weixin.qq.com）
2. 选择你的小游戏账号
3. 进入左侧菜单「流量主」→ 点击「开通」
4. 填写资质信息：
   - 个人主体：上传身份证正反面
   - 企业主体：上传营业执照、对公账户信息
5. 等待审核（通常 1-3 个工作日）
6. 审核通过后即可创建广告位

### 1.2 创建广告位

开通流量主后，需要为每种广告类型创建广告位。

**步骤**：
1. 进入「流量主」→「广告管理」
2. 点击「新建广告位」
3. 选择广告类型（Banner / 激励视频 / 插屏 / 原生模板）
4. 填写广告位名称（方便后续管理）
5. 点击「确定」生成广告位 ID（adUnitId）

**广告位 ID 格式**：
```
adunit-xxxxxxxxxxxxxxxx
```
这是一个唯一的标识符，用于在代码中指定广告位。

### 1.3 开发环境测试

在开通流量主之前，可以使用微信官方提供的测试广告位进行开发测试。

**测试广告位 ID**：
```typescript
const TEST_AD_UNITS = {
    banner: "adunit-0000000000000000",
    rewardedVideo: "adunit-0000000000000001",
    interstitial: "adunit-0000000000000002",
};
```

> **注意**：测试广告位不会产生真实收益，正式发布前必须替换为真实广告位 ID。

## 二、广告类型介绍

### 2.1 Banner 广告

**特点**：
- 固定在屏幕某个位置展示
- 自动刷新（通常 30-60 秒）
- 收益相对较低，但展示时长长

**适用场景**：
- 主界面底部
- 商城界面
- 排行榜界面

**用户体验**：一般（可能遮挡部分内容）

### 2.2 激励视频广告

**特点**：
- 用户主动触发
- 需要看完才能获取奖励
- 收益最高

**适用场景**：
- 获得游戏货币（金币、钻石）
- 复活功能
- 加速功能
- 解锁特殊关卡

**用户体验**：好（用户主动选择观看）

### 2.3 插屏广告

**特点**：
- 全屏展示，视觉冲击强
- 展示时机可控
- 收益中等

**适用场景**：
- 关卡结束
- 返回主界面
- 打开商城

**用户体验**：较差（可能打断用户操作）

### 2.4 原生模板广告

**特点**：
- 可自定义样式
- 融入界面设计
- 收益中等

**适用场景**：
- 商城列表
- 活动推荐
- 信息流

**用户体验**：好（与界面风格一致）

## 三、接入步骤

### 3.1 添加 AdManager 到项目

将 `AdManager.ts` 文件放置到项目的合适位置，例如：
```
assets/scripts/managers/AdManager.ts
```
或
```
docs/wechat-ads/AdManager.ts
```

### 3.2 初始化广告

在游戏启动时（如 `GameManager.onLoad()`）初始化广告：

```typescript
import { AdManager } from './path/to/AdManager';

export class GameManager extends Component {
    onLoad() {
        // 配置广告位 ID
        AdManager.getInstance().configure({
            banner: "adunit-your-banner-id",
            rewardedVideo: {                    // 按位置配置激励视频广告位
                "gold": "adunit-your-rewarded-gold",
                "revive": "adunit-your-rewarded-revive",
                "speedUp": "adunit-your-rewarded-speed",
            },
            interstitial: "adunit-your-interstitial-id",
        });
    }
}
```

### 3.3 显示广告

**Banner 广告**：
```typescript
// 显示 Banner
AdManager.getInstance().showBannerAd();

// 隐藏 Banner（如进入游戏界面）
AdManager.getInstance().hideBannerAd();

// 销毁 Banner（如离开界面）
AdManager.getInstance().destroyBannerAd();
```

**激励视频广告**（按位置指定广告位）：
```typescript
// "gold" 位置使用配置中 "gold" 对应的广告位
AdManager.getInstance().showRewardedVideoAd(
    "gold",                         // 位置标识
    () => {
        // 用户看完广告，给予奖励
        console.log("奖励用户");
    },
    (err) => {
        // 广告加载失败
        console.error("广告失败", err);
    }
);
```

**插屏广告**：
```typescript
AdManager.getInstance().showInterstitialAd();
```

### 3.4 销毁广告

在游戏退出或不需要广告时，销毁广告以释放资源：

```typescript
onDestroy() {
    AdManager.getInstance().destroyAllAds();
}
```

## 四、按位置配置广告位

### 4.1 为什么按位置配置？

按位置（场景）配置不同的广告位可以：
1. **精准投放**：不同场景使用不同的广告位，便于在流量主后台分析各场景收益
2. **灵活控制**：可以单独调整某个位置的广告策略
3. **A/B 测试**：不同位置可以使用不同的广告位进行 A/B 测试
4. **清晰归因**：每个位置的收益来源清晰可查

### 4.2 如何按位置配置？

```typescript
AdManager.getInstance().configure({
    rewardedVideo: {
        "gold": "adunit-rewarded-gold",      // 看视频得金币
        "revive": "adunit-rewarded-revive",   // 复活功能
        "speedUp": "adunit-rewarded-speed",    // 加速功能
        "unlock": "adunit-rewarded-gold",     // 可复用同一广告位
    },
});
```

### 4.3 按位置调用广告

```typescript
// 看视频得金币（使用 "gold" 对应的广告位）
AdManager.getInstance().showRewardedVideoAd(
    "gold",                                   // 位置标识
    () => { console.log("获得金币"); },
    (err) => { console.error("广告失败", err); }
);

// 复活功能（使用 "revive" 对应的广告位）
AdManager.getInstance().showRewardedVideoAd(
    "revive",
    () => { console.log("玩家复活"); }
);
```

### 4.4 工作流程

```
用户点击"看视频得金币"
        ↓
调用 showRewardedVideoAd("gold", onReward)
        ↓
查找 "gold" 对应的广告位 ID
        ↓
找到 adunit-rewarded-gold
        ↓
显示该广告位的激励视频
        ↓
用户看完 → 执行 onReward 回调
用户关闭 → 不执行奖励
```

## 五、审核注意事项

### 5.1 微信审核要求

**必须遵守的规则**：
1. 不得强制用户观看广告
2. 广告不得遮挡功能按钮
3. 激励视频必须给予明确的奖励提示
4. 广告位名称需准确描述
5. 不得在启动页显示广告
6. 不得在加载页显示广告

### 5.2 激励视频规范

**正确做法**：
```typescript
// 用户主动点击按钮
<button>看视频获得 100 金币</button>

// 明确提示奖励内容（指定位置 "gold"）
AdManager.getInstance().showRewardedVideoAd(
    "gold",                          // 位置标识
    () => { /* 给予 100 金币 */ },
    (err) => { /* 处理失败 */ }
);
```

**错误做法**：
```typescript
// 强制观看才能继续游戏
if (!hasWatchedAd) {
    showRewardedVideoAd();  // 不允许跳过
}
```

### 5.3 Banner 广告规范

- 不得遮挡关闭按钮
- 不得遮挡重要操作按钮
- 建议放在底部或顶部，不影响操作

### 5.4 插屏广告规范

- 不得频繁弹出（建议间隔至少 2 分钟）
- 不得在游戏进行中弹出
- 建议在自然切换点展示（如关卡结束）

## 六、常见问题

### 6.1 广告加载失败怎么办？

**可能原因**：
1. 广告位 ID 错误
2. 流量主未开通或审核未通过
3. 网络问题
4. 广告未填充（errCode: 1004）

**解决方法**：
- 检查广告位 ID 是否正确
- 确认流量主已开通
- 按位置配置多广告位，便于问题定位
- AdManager 已内置失败重试和降级处理

### 6.2 如何测试广告功能？

使用测试广告位 ID 进行开发测试：

```typescript
const TEST_AD_UNITS = {
    banner: "adunit-0000000000000000",
    interstitial: "adunit-0000000000000003",
    rewardedVideo: {
        "gold": "adunit-0000000000000001",
        "revive": "adunit-0000000000000002",
    },
};
```
### 6.3 非微信环境如何调试？

AdManager 已处理非微信环境的情况：
```typescript
// 非微信环境下，激励视频会直接执行回调
if (typeof wx === 'undefined') {
    onReward();  // 直接给奖励
    return;
}
```

### 6.4 如何提升广告收益？

**核心策略**：
1. **激励视频是核心**：占收入的 70-80%，与玩法深度结合
2. **合理设置奖励**：奖励要有吸引力，但不要过于丰厚
3. **控制展示频率**：避免用户反感
4. **按位置配置广告位**：不同场景使用不同广告位，便于数据分析和优化

## 七、总结

### 接入流程总结

```
1. 开通流量主
   ↓
2. 创建广告位，获取广告位 ID
   ↓
3. 将 AdManager.ts 添加到项目
   ↓
4. 初始化广告（配置广告位 ID）
   ↓
5. 在合适的时机展示广告
   ↓
6. 测试广告功能
   ↓
7. 提交审核
   ↓
8. 发布上线，监控收益数据
```

### 关键要点

- ✅ 激励视频支持按位置配置多广告位，精准投放
- ✅ 广告展示要自然，不影响用户体验
- ✅ 做好错误处理和降级方案
- ✅ 遵守微信审核规范，避免被拒
- ✅ 监控各位置广告数据，持续优化

祝你的游戏广告收益越来越高！💰
