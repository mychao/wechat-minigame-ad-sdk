# 微信小游戏广告优化策略

## 一、收益优化目标

广告收益核心公式：
```
广告收益 = 展示量 × 点击率 × eCPM / 1000
```

优化目标：提升展示量、点击率和 eCPM。

---

## 二、激励视频优化（核心收入来源）

### 2.1 激励视频占比

激励视频通常占广告收入的 **70-80%**，是优化的重点。

### 2.2 接入场景建议

| 场景 | 奖励内容 | 每日次数限制 | 优先级 |
|------|---------|-------------|--------|
| 看视频获得金币 | 100-500 金币 | 5-10 次 | ⭐⭐⭐⭐⭐ |
| 战斗复活 | 原地复活 | 3-5 次 | ⭐⭐⭐⭐⭐ |
| 加速功能 | 加速 2-5 倍 | 3-5 次 | ⭐⭐⭐⭐ |
| 解锁关卡 | 解锁特殊关卡 | 1-3 次 | ⭐⭐⭐ |
| 额外抽奖 | 多抽一次 | 3-5 次 | ⭐⭐⭐⭐ |

### 2.3 按位置配置广告位策略

为不同场景配置不同的广告位，实现精准投放：

```typescript
// 按位置配置激励视频广告位
AdManager.getInstance().configure({
    rewardedVideo: {
        "gold": "adunit-gold",         // 看视频得金币
        "revive": "adunit-revive",     // 复活功能
        "speedUp": "adunit-speedup",   // 加速功能
    },
});
```

**收益提升效果**：
- 精准归因：各场景收益来源清晰
- 灵活调整：可单独优化某个位置的广告策略
- A/B 测试：不同位置可使用不同广告位测试

### 2.4 展示频率控制

```typescript
export class RewardVideoStrategy {
    private todayShowCount: number = 0;
    private lastShowTime: number = 0;
    private readonly MAX_DAILY_COUNT = 15;  // 每日最多 15 次
    private readonly MIN_INTERVAL = 30000;  // 间隔至少 30 秒

    canShow(): boolean {
        // 检查每日次数
        if (this.todayShowCount >= this.MAX_DAILY_COUNT) {
            return false;
        }

        // 检查时间间隔
        const now = Date.now();
        if (now - this.lastShowTime < this.MIN_INTERVAL) {
            return false;
        }

        return true;
    }

    onShowSuccess() {
        this.todayShowCount++;
        this.lastShowTime = Date.now();
    }
}
```

## 三、Banner 广告优化

### 3.1 展示位置

**推荐位置**：
- 主界面底部（固定）
- 商城界面底部
- 排行榜界面底部

**不推荐位置**：
- 游戏进行中（影响操作）
- 按钮附近（误触率高）
- 屏幕中央（遮挡内容）

### 3.2 展示时长

- 同一界面持续展示，不要频繁创建/销毁
- 离开界面时销毁
- 进入界面 1 秒后显示（提升用户体验）

### 3.3 Banner 收益占比

Banner 广告通常占收入的 **10-15%**，主要起辅助作用。

---

## 四、插屏广告优化

### 4.1 展示时机

**推荐时机**：
- 关卡完成（延迟 2 秒显示）
- 返回主界面
- 打开商城

**不推荐时机**：
- 游戏进行中
- 用户操作中
- 频繁弹出（间隔 < 2 分钟）

### 4.2 频率控制

```typescript
export class InterstitialStrategy {
    private lastShowTime: number = 0;
    private readonly MIN_INTERVAL = 120000; // 至少间隔 2 分钟
    private readonly MAX_PER_HOUR = 4;      // 每小时最多 4 次

    canShow(): boolean {
        const now = Date.now();
        return (now - this.lastShowTime) >= this.MIN_INTERVAL;
    }
}
```

### 4.3 插屏收益占比

插屏广告通常占收入的 **10-15%**。

## 五、用户分层策略

### 5.1 用户分类

| 用户类型 | 特征 | 广告策略 |
|---------|------|---------|
| 付费用户 | 已购买去广告或充值 | 减少广告展示 |
| 活跃免费用户 | 每日活跃，未付费 | 正常展示广告 |
| 新用户 | 注册 3 天内 | 减少广告，避免流失 |
| 流失风险用户 | 活跃度下降 | 增加激励视频，提供奖励 |

### 5.2 实现示例

```typescript
export class AdStrategyManager {
    shouldShowAd(user: User): boolean {
        // 付费用户：不展示广告
        if (user.isPaidUser) {
            return false;
        }

        // 新用户（3 天内）：减少 50% 广告
        if (user.daysSinceRegister <= 3) {
            return Math.random() < 0.5;
        }

        // 普通用户：正常展示
        return true;
    }

    getRewardAmount(user: User): number {
        // 新用户：奖励翻倍，提升参与度
        if (user.daysSinceRegister <= 7) {
            return 200;  // 普通用户 100
        }
        return 100;
    }
}
```

## 六、A/B 测试建议

### 6.1 测试维度

| 测试项 | 方案 A | 方案 B | 评估指标 |
|--------|--------|--------|----------|
| 激励视频奖励 | 100 金币 | 200 金币 | 观看率、eCPM |
| 插屏频率 | 每 2 分钟 | 每 5 分钟 | 留存率、收益 |
| Banner 位置 | 底部 | 顶部 | 点击率、留存率 |
| 复活广告 | 必看 | 可选 | 使用率、留存率 |

### 6.2 数据上报

```typescript
export class AdAnalytics {
    // 记录广告展示
    trackAdShow(adType: string, adUnitId: string) {
        // 上报到数据分析平台

    }

    // 记录广告点击
    trackAdClick(adType: string) {
        console.log(`[Analytics] 广告点击: ${adType}`);
    }

    // 记录激励视频完成
    trackRewardComplete(rewardType: string, amount: number) {
        console.log(`[Analytics] 激励视频完成: ${rewardType}, ${amount}`);
    }

    // 记录广告失败
    trackAdError(adType: string, errCode: number) {
        console.log(`[Analytics] 广告失败: ${adType}, errCode: ${errCode}`);
    }
}
```
